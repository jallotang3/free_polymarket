### 亏损单共性

1. **低波动环境**（波动↓）占比 >80%
2. **震荡市**（趋势震荡）或**弱趋势**（可信<0.60）
3. **Gap不足**（<0.10%）或**Gap≈0**
4. **中等赔率**（0.75~0.85）在分2/分3不够强

## 三、优化方案

### 优化1：低波动环境收紧路径2

**数据依据**：亏损单中80%发生在"波动↓"环境。

```python
# strategy.py _eval_odds_only()
def _eval_odds_only(...):
    # 新增：低波动环境提高赔率门槛
    vol_level = ... # 从 market_context 获取
    if vol_level == "low":
        # 低波动时，路径2最低赔率从0.78提高到0.85
        if odds_px < 0.85:
            logger.debug("低波动环境拦截: odds=%.2f < 0.85", odds_px)
            return None
```

**预期效果**：过滤掉#319类"gap≈0 + 赔率0.78 + 低波动"的弱信号。

### 优化2：震荡市提高可信度门槛

**数据依据**：#326/#327在震荡市（可信0.48）下单后反向。

```python
# strategy.py _eval_odds_only()
# 分2/分3 震荡市（可信<0.55）时，要求更高赔率或更大gap
if minute <= 3 and signal_confidence < 0.55:
    # 震荡市要求：赔率≥0.85 或 gap同向≥0.10%
    if odds_px < 0.85 and not (gap_same_dir and abs(gap) >= 0.10):
        logger.debug(
            "震荡市拦截: conf=%.2f odds=%.2f gap=%.3f%%",
            signal_confidence, odds_px, gap
        )
        return None
```

**预期效果**：震荡市只接受"高赔率(≥0.85)"或"强gap(≥0.10%)"的双确认信号。

---

### 优化3：分3高赔率双确认加强

**现状问题**：分3高赔率（>0.85）要求conf≥0.60，但#319案例显示gap≈0时仍不可靠。

```python
# strategy.py _eval_odds_only()
# 分3 高赔率（>0.85）双确认：从"gap≥0.05%"提高到"gap≥0.10%"
if minute >= 3 and odds_px > 0.85:
    gap_ok  = gap_same_dir and abs(gap) >= 0.10  # 从0.05提高到0.10
    conf_ok = signal_confidence >= 0.60
    if not (gap_ok and conf_ok):
        logger.debug(
            "分3高赔率双确认拦截: odds=%.2f gap=%.3f%%(same=%s) conf=%.2f",
            odds_px, gap, gap_same_dir, signal_confidence
        )
        return None
```

**预期效果**：分3高赔率必须有"gap≥0.10% + conf≥0.60"双重保障。

---

### 优化4：路径2 EV门槛动态调整

**现状问题**：低波动+震荡市时，即使EV达标，信号质量仍不足。

```python
# strategy.py _eval_odds_only()
# EV门槛根据市场环境动态调整
base_ev = gp["min_ev"]
if vol_level == "low":
    ev_threshold = base_ev * 2.0  # 低波动：EV门槛翻倍
elif signal_confidence < 0.50:
    ev_threshold = base_ev * 1.5  # 低可信度：EV门槛×1.5
elif _local_hour == 23:
    ev_threshold = base_ev * 1.75  # 23点高波动时段
else:
    ev_threshold = base_ev

if ev < ev_threshold:
    logger.debug(
        "EV拦截: ev=%.4f < %.4f (vol=%s conf=%.2f)",
        ev, ev_threshold, vol_level, signal_confidence
    )
    return None
```

**预期效果**：低波动环境下，只接受EV≥0.16（greed=5时base_ev=0.08×2）的超强信号。

### 优化5：贪婪指数调整建议

**当前设置**：GREED_INDEX=5（平衡档）
- min_odds_path2=0.78

**建议调整**：降低到GREED_INDEX=4（偏保守）

1. **过度拟合风险**：优化基于137笔样本，需更多数据验证
