### 1.1 已验证的核心结论

通过从 Binance 获取 **7 天 × 10,080 条 1 分钟 K 线**，在 **2,015 个 5 分钟窗口**上完整回测后，得到以下关键结论：

| 策略类型 | 历史胜率 | 结论 |
|----------|---------|------|
| 动量策略（过去60秒方向） | **46.1%** | ❌ 反而低于随机，不能用 |
| 动量策略（ATR归一化）     | **52.5%** | ❌ Kelly公式仍判定负期望 |
| 均值回归策略             | **50.5%** | ❌ 略优于随机但覆盖不了手续费 |
| **末期定价套利（第4分钟）** | **96.8%** | ✅ 高胜率，**关键在入场价格** |

**最重要的发现**：BTC 5 分钟方向是**近似随机游走**，动量/均值回归类策略均无效。真正有效的唯一策略是**末期定价套利**——利用真实胜率与 Polymarket 市场定价之间的差距。

## 二、优化后的核心策略：末期定价套利

> 放弃动量/均值回归策略，聚焦于唯一经验证的有效策略

### 2.1 策略逻辑

```
触发条件（同时满足）：
  1. 距窗口结束 ≤ 90 秒（进入末期阶段）
  2. BTC 实时价格与窗口开盘价的差距 ≥ 0.10%
  3. Polymarket 对应方向的赔率 < 0.93（保证足够利润空间）
  4. 市场深度（asks/bids）充足（防止滑点）

入场逻辑：
  - 计算「理论胜率」= f(gap%, 剩余时间, BTC实时波动率)
  - 若 理论胜率 > Polymarket赔率 + 0.03（3%安全边际）
  - → 下限价单，买入对应方向

仓位管理：
  - 使用 Kelly 公式 × 0.5（半 Kelly）
  - 单笔最大 5% 总资金
  - 期望值 < 0.01/单位时 → 不操作
```

### 2.2 入场时机精确化

```
时间轴（5分钟窗口内）：

0:00 ─── 0:30 ─── 1:00 ─── 2:00 ─── 3:00 ─── 3:30 ─── 4:00 ─── 4:30 ─── 5:00
  开盘      观察                                  初步评估         最优入场      结算
                                                  (3min gap)       (4min gap)
                                               胜率 ≥ 93%        胜率 ≥ 97%

  最优入场窗口：第 3:30 ~ 4:30 分钟
  此时：
    - 胜率已充分显现（86%~97%）
    - 距结算还有 30~90 秒，赔率尚未完全收敛到 1.0
    - 赔率通常仍在 0.80~0.93，存在套利空间
```

### 2.3 Chainlink 价格优势

Polymarket 使用 **Chainlink BTC/USD 数据流**结算，而非 Binance spot。关键：

- Chainlink 价格 ≠ Binance 价格（存在微小偏差）
- 但两者高度相关，Binance 实时价格可作为 Chainlink 的**领先指标**
- 实现方案：订阅 Binance WebSocket 1s K线 → 推算 Chainlink 当前位置 → 与 Polymarket 赔率对比

## 五、风险管理

### 5.1 资金规则

```
单次最大下注 = 总资金 × min(半Kelly, 5%)
日最大亏损   = 总资金 × 15%  → 触发自动停机
连亏 5 次    → 暂停 1 小时，人工复盘
月最大亏损   = 30%  → 策略审查
```

### 5.2 期望收益估算（保守）

```
假设：
  - 每日 BTC 交易时段约 16 小时
  - 每小时 12 个窗口，16小时 = 192 个窗口/天
  - gap ≥ 0.10% 的比例 ≈ 32.8%（历史数据）
  - 触发次数 ≈ 192 × 32.8% ≈ 63 次/天
  - 实际入场（扣除赔率不划算的窗口）≈ 30-40 次/天
  - 平均期望收益 ≈ 0.03/单位（保守）
  - 单次投入 $50（总资金 $1000 × 5%）

日期望收益 = 35 次 × $50 × 0.03 = $52.5/天
月期望收益 ≈ $1,575（在策略持续有效的前提下）

⚠️ 这是数学期望，不是保证收益。实盘验证后才能确认。
```

### 5.3 主要风险

| 风险 | 影响 | 对策 |
|------|------|------|
| 市场已完全定价（折扣=0） | 策略失效 | Phase 1 验证后再入场 |
| 专业机器人竞争 | 套利窗口缩短到 < 1 秒 | 优化网络延迟；降低规模 |
| Chainlink 极端更新 | 最后 1 秒价格逆转 | gap ≥ 0.10% 才操作，逆转率仅 3.2% |
| Polygon gas 费 | 每笔 ~$0.01-0.03 | 小额订单时注意成本 |
| 私钥安全 | 资金损失 | 专用小额钱包，硬件签名 |
