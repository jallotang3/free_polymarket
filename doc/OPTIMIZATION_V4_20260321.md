## 一、核心发现：策略过于保守，信号被过度过滤

### 当前策略状态（禁用 TWR=0.860 后）

| 指标 | 数值 |
|------|------|
| 实盘累计交易 | 117 笔 |
| 实盘胜率 | 76.9% |
| 实盘 PnL | -$22.20 |
| 每日信号数（当前规则） | ~5-8 次 |
| 每日信号数（852窗口回测） | 23.7 次（gap≥0.10% + odds<0.80） |

**问题：禁用 TWR=0.860 后确实消除了亏损来源，但也砍掉了大量可盈利的机会。**

### 852 窗口回测数据揭示的关键事实

**赔率阈值 vs 胜率 vs 信号频率（分3+）：**

| 赔率阈值 | 窗口数 | 胜率 | 次/天 | 真实EV |
|---------|--------|------|-------|--------|
| ≥0.72 | 472 | **92.4%** | 135 | +0.0406 |
| ≥0.75 | 432 | 93.5% | 124 | +0.0382 |
| **≥0.78** | **400** | **94.8%** | **114** | **+0.0399** |
| ≥0.80 | 379 | 95.0% | 108 | +0.0357 |
| ≥0.85 | 305 | 96.4% | 87 | +0.0285 |
| ≥0.92 | 189 | 98.4% | 54 | +0.0188 |

**最优平衡点是 odds ≥ 0.78**：胜率 94.8% + EV=0.0399 + 每天 114 次信号，兼顾盈利和风险。

**关键组合（新候选策略）：**
```
odds ≥ 0.78 + gap 同向 ≥ 0.05%（分3+）：
  胜率 = 94.7%  信号 = 91.2 次/天
  → 这是当前策略应该采用的主力信号区间
```

## 二、4coinsbot 对比分析

### 4coinsbot 策略核心（Late Entry V3）

| 参数 | 4coinsbot | 我们 | 评价 |
|------|----------|------|------|
| 入场窗口 | **最后4分钟（240s）** | 分1~分4（60~270s） | 类似 |
| 最低赔率 | 无最低要求 | 当前 0.85 | 他们更激进 |
| 赔率差筛选 | `min_confidence=0.30`（赔率差>30%） | 无直接等价 | 简洁有效 |
| 最高入场价 | `price_max=0.92` | 无上限 | 值得借鉴 |
| 下单频率 | 每7秒最多一次 | 每窗口最多一次 | 他们更频繁 |
| 仓位管理 | 固定合约数（8/10/12）时间分级 | Kelly 公式 | 各有优劣 |
| 止损 | ✅ 固定金额止损（-$12/币种） | ❌ 无 | 值得加入 |
| Flip-Stop | ✅ 赔率反转保护（跌破0.48卖出） | ❌ 无 | 值得加入 |
| 多币种 | BTC/ETH/SOL/XRP 并行 | 仅 BTC | 可扩展 |
| WebSocket | ✅ 实时推送 | ❌ 5s 轮询 | 他们更快 |

### 4coinsbot 值得我们学习的 3 点

1. **`min_confidence = 0.30`（赔率差筛选）**
   - 只需 `abs(up_ask - down_ask) > 0.30` 即可入场
   - 对应赔率约 ≥ 0.65，比我们的 0.85 宽松很多
   - 但他们的 `price_max=0.92` 限制了高赔率，避免利润太薄

2. **Flip-Stop（赔率反转保护）**
   - 当持仓方向的赔率跌破 0.48（变成非主导方），立即卖出止损
   - 这正是我们之前讨论的"延迟止损"的变体

3. **时间分级仓位（而非 Kelly）**
   - 时间越少（越接近结算）下越大：8 → 10 → 12 合约
   - 简洁直观，避免 Kelly 复利失控

### 4coinsbot 的不足

1. **没有使用 Chainlink 链上价格**——纯靠赔率差做信号，精度低于我们的 gap 分析
2. **没有历史趋势/背景过滤**——我们的 MarketContext 有一定价值
3. **固定合约数**——不根据胜率和 EV 动态调整，适合大资金

## 三、贪婪指数（Greed Index）设计

### 设计思路

贪婪指数 `GREED_INDEX` 取值 1~10，控制策略的冒险程度。
数值越高，入场条件越宽松，信号越多，但胜率越低。

```
GREED_INDEX=1 → 极保守：只做 gap≥0.20% + odds≥0.92（几乎不输，但机会极少）
GREED_INDEX=5 → 平衡：   gap≥0.10% + odds≥0.78 + gap同向（默认推荐）
GREED_INDEX=10→ 激进：   odds≥0.65 + 不限gap（频率高但胜率约85%）
```

### 参数映射表

| 贪婪指数 | 路径2 赔率下限 | 路径3 gap 下限 | EV门槛 | 分钟限制 | 预期胜率 | 预期信号/天 |
|---------|-------------|-------------|---------|---------|---------|-----------|
| 1 | 0.92 | 0.20% | 0.15 | 分4 only | ~98% | ~3 |
| 2 | 0.90 | 0.15% | 0.12 | 分3+ | ~97% | ~8 |
| 3 | 0.88 | 0.12% | 0.10 | 分3+ | ~96% | ~15 |
| 4 | 0.85 | 0.10% | 0.10 | 分3+ | ~95% | ~25 |
| **5** | **0.78** | **0.10%** | **0.08** | **分3+** | **~94%** | **~40** |
| 6 | 0.75 | 0.08% | 0.06 | 分2+ | ~92% | ~60 |
| 7 | 0.72 | 0.05% | 0.05 | 分2+ | ~90% | ~80 |
| 8 | 0.72 | 无 | 0.04 | 分2+ | ~88% | ~100 |
| 9 | 0.65 | 无 | 0.03 | 分1+ | ~85% | ~130 |
| 10 | 0.55 | 无 | 0.02 | 分1+ | ~80% | ~160 |

### 推荐值

- **小资金（<$50）：GREED_INDEX=5~6**（需要一定频率才能积累收益）
- **中资金（$50~200）：GREED_INDEX=4~5**
- **大资金（>$200）：GREED_INDEX=3~4**（降低风险，大单不适合频繁交易）
